API V2 Service Accounts

API Credentials and Keys

A V2 service account is required to use the V2 API. A service account is a ClientId, permission set, and secret set. A service account may define several valid OIDC credentials.

What Changed?

Access

API V1 service accounts are linked to a Starlink user and allow API access for any Starlink account the user was on.

API V2 service accounts are linked to a Starlink account and exist independently of the user that created them. V2 service accounts will remain valid until disabled, even if the user that created the service account is no longer on the account.

Roles (Role Based Access Control)

API V2 service accounts can only access endpoints according to their assigned permissions. Review the API reference to understand which endpoint requires which permission.

Who can make V2 Service Accounts

Creating or editing API V2 service accounts can be done by any user who has the "Admin" or "Service Account Management" role. Users can select which permissions an API V2 service account will have - up to the user's individual permissions.

Any user with permission to create or update V2 service accounts also has permission to disable previously created V2 service accounts. This permission should be given to trusted users only.

Any user with the role Admin, Technical, Technical Read Only, Technical Configuration, or Service Account Management can see the list of existing V2 service accounts.

Although service account secrets (aka keys) can be deleted, the overall service account structure can only be disabled. This design is to preserve traceability.
